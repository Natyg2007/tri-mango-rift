Tri-Mango Rift
Play Systems Notes

Playable browser game prototype

Shift between three rifts to survive the orchard maze.

Collect matching mango cores, dodge phase-linked hazards, and use the three-state rift system to route through a moving arena. Built as a compact work sample for 2D engine scripting, gameplay state, input, collision, UI, and polish.

What it demonstrates

Game systems in the prototype

01

Three-state gameplay

The active rift changes which collectibles are valid, which hazards are dangerous, the arena tint, and the player trail feedback.

02

Entity loop and collision

Moving sentries, orbiting cores, player bounds, circle collisions, score state, timer state, and restart flow run from a clean update loop.

03

Portable engine logic

The project maps cleanly to Godot, Defold, Solar2D, Love2D, PyGame, MonoGame, Unity, Unreal, O3DE, Stride, or similar game engines.
